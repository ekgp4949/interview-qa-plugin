# 기술 면접 준비 스킬

## 면접 질문 유형

### 1. 개념 설명형
> "X가 무엇인지 설명해주세요"

**답변 구조**:
1. 한 문장 정의
2. 핵심 특징 3가지
3. 실제 사용 예시
4. 장단점 (선택)

### 2. 비교형
> "A와 B의 차이점은?"

**답변 구조**:
1. A의 핵심 특징
2. B의 핵심 특징
3. 주요 차이점 (표 형태 추천)
4. 각각 언제 사용하는지

### 3. 문제 해결형
> "X 문제를 어떻게 해결하시겠습니까?"

**답변 구조**:
1. 문제 재정의 (이해 확인)
2. 가능한 접근 방법들
3. 선택한 방법과 이유
4. 트레이드오프 설명
5. 추가 고려사항

### 4. 경험 기반형
> "X를 구현해본 경험이 있나요?"

**답변 구조 (STAR 기법)**:
- **S**ituation: 상황 설명
- **T**ask: 해결해야 할 과제
- **A**ction: 취한 행동
- **R**esult: 결과 및 배운 점

### 5. 설계형
> "X 시스템을 설계해주세요"

**답변 구조**:
1. 요구사항 확인 (질문하기)
2. 높은 수준 설계
3. 핵심 컴포넌트 상세
4. 데이터 흐름
5. 확장성/장애 대응

## 카테고리별 핵심 개념

### CS 기초

#### 자료구조
| 자료구조 | 시간복잡도(평균) | 사용 시점 |
|----------|------------------|-----------|
| Array | O(1) 접근, O(n) 삽입 | 인덱스 접근 빈번 |
| LinkedList | O(n) 접근, O(1) 삽입 | 삽입/삭제 빈번 |
| HashMap | O(1) 접근/삽입 | 키-값 매핑 |
| Tree | O(log n) | 정렬된 데이터 |
| Graph | 다양 | 관계 표현 |

#### 알고리즘
- **정렬**: QuickSort(평균 O(n log n)), MergeSort(안정 정렬)
- **탐색**: Binary Search(정렬된 배열), DFS/BFS(그래프)
- **DP**: 중복 부분 문제 + 최적 부분 구조

### 아키텍처

#### SOLID 원칙
- **S**: Single Responsibility - 하나의 책임만
- **O**: Open/Closed - 확장에 열림, 수정에 닫힘
- **L**: Liskov Substitution - 하위 타입 대체 가능
- **I**: Interface Segregation - 인터페이스 분리
- **D**: Dependency Inversion - 추상화에 의존

#### 디자인 패턴 (자주 출제)
1. **Singleton**: 인스턴스 하나만
2. **Factory**: 객체 생성 캡슐화
3. **Observer**: 상태 변화 알림
4. **Strategy**: 알고리즘 교체 가능
5. **Decorator**: 동적 기능 추가

### 보안

#### OWASP Top 10 (2021)
1. Broken Access Control
2. Cryptographic Failures
3. Injection
4. Insecure Design
5. Security Misconfiguration
6. Vulnerable Components
7. Authentication Failures
8. Data Integrity Failures
9. Logging Failures
10. SSRF

#### 보안 체크리스트
- [ ] 입력값 검증 (서버 사이드)
- [ ] 출력 인코딩 (XSS 방지)
- [ ] 파라미터화된 쿼리 (SQL Injection 방지)
- [ ] CSRF 토큰
- [ ] HTTPS 적용
- [ ] 비밀번호 해싱 (bcrypt, argon2)
- [ ] 민감 데이터 암호화

### 성능

#### 최적화 전략
1. **캐싱**: 메모리 > 디스크 > 네트워크
2. **인덱싱**: 적절한 DB 인덱스
3. **비동기**: 블로킹 작업 분리
4. **압축**: 데이터 전송량 감소
5. **CDN**: 정적 자원 분산

#### 병목 지점 식별
- CPU bound: 계산 집약적
- I/O bound: 네트워크/디스크
- Memory bound: 메모리 부족

## 면접 팁

### DO
- 모르면 솔직히 인정
- 생각하는 과정을 말로 설명
- 질문하여 요구사항 명확히
- 트레이드오프 언급
- 실제 경험 연결

### DON'T
- 암기한 듯한 답변
- 너무 길게 설명
- 질문 의도 파악 없이 답변
- 부정적 경험만 언급
- 기술 깊이 없이 나열만
