# QA 생성 가이드라인

## 질문 작성 원칙

### DO
- "왜?"와 "어떻게?"를 묻는 질문
- 트레이드오프를 설명할 수 있는 질문
- 실제 코드/프로젝트와 연관된 구체적 질문
- 꼬리 질문으로 깊이 있는 대화 유도
- 난이도를 명확히 표시

### DON'T
- 단순 암기형 질문 (정의만 묻는 질문)
- 너무 일반적이거나 광범위한 질문
- 예/아니오로 답할 수 있는 질문
- 코드와 무관한 이론적 질문만

## 답변 작성 원칙

### 구조
1. 핵심 답변 (1-2문장)
2. 상세 설명
3. 실제 예시 또는 코드
4. 관련 고려사항

### 품질 기준
- 면접관에게 설명하듯 명확하게
- 기술 용어는 설명과 함께
- 트레이드오프 반드시 언급
- 실무 경험과 연결 가능한 답변

## 카테고리 분류 기준

| 카테고리 | 포함 내용 |
|----------|----------|
| CS 기초 | 자료구조, 알고리즘, OS, 네트워크 |
| 아키텍처 | 패턴, 설계 원칙, 시스템 설계 |
| 보안 | 취약점, 인증, 암호화 |
| 성능 | 최적화, 캐싱, 스케일링 |
| 언어/FW | 특정 기술 스택 관련 |

## 난이도 기준

- **★☆☆**: 기본 개념, 주니어 수준
- **★★☆**: 심화 내용, 미들 수준
- **★★★**: 고급 주제, 시니어 수준
