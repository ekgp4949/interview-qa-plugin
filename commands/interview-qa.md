# /interview-qa 명령어

## 개요

Claude Code 세션에서 수행한 작업이나 프로젝트 구조를 분석하여 기술 면접에 활용할 수 있는 QA(Question & Answer) 리스트를 생성합니다.

## 사용법

```bash
# 현재 세션 전체 분석
/interview-qa

# 특정 주제 지정
/interview-qa "React hooks"
/interview-qa "인증 구현"

# 마지막 작업만 분석
/interview-qa --last

# 특정 파일 관련
/interview-qa --file src/auth.ts

# 프로젝트 전체 구조 분석
/interview-qa --project

# 특정 경로만 분석
/interview-qa --project src/
```

## 실행 절차

### 1. 분석 대상 결정

인자에 따라 분석 범위를 결정합니다:

- **인자 없음**: 현재 세션의 모든 대화와 작업 내용
- **주제 지정**: 해당 주제와 관련된 세션 내용만
- **--last**: 가장 최근 작업만
- **--file**: 특정 파일과 관련된 내용만
- **--project**: 프로젝트 전체 구조 분석

### 2. qa-generator 에이전트 호출

분석 대상을 qa-generator 에이전트에 전달하여 QA를 생성합니다.

### 3. QA 파일 저장

생성된 QA를 다음 위치에 저장합니다:

```
~/Desktop/QA/YYYYMMDD_HHMMSS_프로젝트명_주제.md
```

파일명 예시:
- `20260125_143052_claude-test_React_상태관리.md`
- `20260125_150000_my-app_프로젝트_아키텍처.md`

**프로젝트명 결정 규칙**:
- 현재 작업 디렉토리의 폴더명 사용
- 특수문자는 언더스코어(_)로 치환

### 4. 결과 요약 출력

생성된 QA 개수와 카테고리별 분포를 요약하여 출력합니다.

## 생성되는 QA 형식

```markdown
# 기술 면접 QA - {주제}

**생성일**: YYYY-MM-DD HH:MM:SS
**분석 대상**: {세션/프로젝트/파일명}
**총 질문 수**: N개

---

## Q1: {질문}

**카테고리**: CS 기초 | 아키텍처 | 보안 | 성능
**난이도**: ★☆☆ ~ ★★★

### 답변

{상세 답변}

### 꼬리 질문

1. {추가 질문 1}
2. {추가 질문 2}

### 핵심 키워드

`키워드1`, `키워드2`

---

## Q2: ...
```

## 분석 카테고리

| 카테고리 | 분석 내용 |
|----------|----------|
| CS 기초 | 자료구조, 알고리즘, 복잡도, 메모리 |
| 아키텍처 | 디자인 패턴, SOLID, 시스템 설계 |
| 보안 | OWASP, 인증/인가, 암호화 |
| 성능 | 캐싱, 최적화, 비동기 처리 |
| 언어/FW | 사용된 언어/프레임워크 특화 질문 |

## 프로젝트 분석 시 추가 질문

`--project` 옵션 사용 시 다음 질문들이 추가됩니다:

- "이 프로젝트의 전체 아키텍처를 설명해주세요"
- "기술 스택 선택 이유는 무엇인가요?"
- "폴더 구조는 어떤 원칙으로 설계했나요?"
- "주요 모듈 간의 의존성은 어떻게 관리하나요?"
- "테스트 전략은 어떻게 되나요?"

## 팁

1. **세션 중간에 실행**: 특정 작업 완료 후 바로 실행하면 해당 작업에 대한 QA 생성
2. **주제 지정 활용**: 넓은 세션에서 특정 주제만 추출하고 싶을 때 유용
3. **프로젝트 온보딩**: 새 프로젝트 참여 시 `--project`로 구조 파악 및 면접 준비
4. **정기적 복습**: 생성된 QA를 `/review-qa`로 복습하여 기술 면접 대비
